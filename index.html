¿Qué es un repositorio de código?

Como implica la palabra repositorio, es un lugar de almacenamiento. Entonces, ¡un repositorio de código es un lugar para almacenar código! Ahora eso no es especial. Simplemente puede crear una carpeta en su computadora llamada Código fuente y poner todo allí. ¿Cuál es el punto de darle a este lugar de almacenamiento un nombre elegante?

Se crea un repositorio de código para usar sistemas de versiones como se menciona en el capítulo anterior. El repositorio de código contiene el código fuente mientras que el software del sistema de versiones archiva ese código. Puede archivar todos sus archivos en un repositorio, manteniendo cualquier otra versión o archivo, incluso si no los está utilizando en este momento. Los repositorios de código también le brindan una forma de nombrar o etiquetar las diferentes versiones, manteniendo registros de los cambios dentro del mismo proyecto.

Repositorios locales, remotos y descentralisados

Locales
  • Trabajas desde tu máquina, todo lo haces tú 😎.
  • No necesitas tener tu cuenta en GitHub basta con tener Git.
  • Básicamente lo usas cuando estas comenzando en este mundo de Git.
Remotos
  • Es necesario tener cuenta de GitHub o otro Software de Control de Versiones.
  • Si ya conoces Git no se te hace tan difícil entender el funcionamiento del repositorio remoto.
  • Tienes mas seguro tu código por si se te malogra tu PC ya que se encuentra en la nube.
